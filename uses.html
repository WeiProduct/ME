Stack Updated May 2026

What I ship with.

The tools and services I use to ship AI consumer apps at WeiProduct. Updated as the stack changes.

The studio is one person. The lever is reuse — an in-house toolkit (SwiftUI components, auth, IAP, telemetry, server proxy template) cuts average time from new-app kickoff to App Store submission by roughly sixty percent. Everything below is part of that toolkit or directly supports it.

iOS

  • Swift & SwiftUI — every shipped app. Declarative UI is the biggest single multiplier when one person designs, builds, and ships.
  • Xcode + App Store Connect — full build → review → release loop, done manually per app. I have not invested in CI for iOS yet; volume hasn’t justified it.

Web & backend

  • TypeScript & Node.js — marketing sites, support pages, and the chat proxy in front of OpenAI for this site.
  • Python — internal tooling: data cleanup, App Store metadata wrangling, prompt evals.
  • Vercel — every web property and serverless function. Pay-as-you-go matches a single-person studio’s cost curve better than dedicated infra.

AI / ML

  • OpenAI — primary model provider for chat and structured output. gpt-4o-mini handles most production traffic; cost-effective for consumer use.
  • Whisper — speech-to-text, powering AI Voice Notes.
  • Anthropic — secondary provider; useful when a model needs more context retention or a different prose voice.
  • No training, no fine-tunes — I lean on hosted models. The value of a consumer AI product lives in product surfaces, not in the weights underneath.

Operating principle

The stack is conservative on purpose. Every layer I add is a layer I have to maintain across seventeen apps. Default to fewer dependencies, simpler primitives, and hosted services. Replace something only when it’s a clear bottleneck — never because it’s shiny.

— Wei Fu · May 2026
